Why this doc exists. The messaging system locks the language. This doc locks the schedule: which post goes on which day, in which order, with what kind of engagement work in between. Summary

Four pre-launch posts. Each is drawn verbatim from the locked messaging system. Posted on a staggered cadence (one every 4-7 days) so each gets its own attention window. The first post is pinned. Between scheduled posts, the founder engages with adjacent threads in the niche (Obsidian, MCP, AI tools, voice tools) but does not publish new positioning copy outside this set.

  • Post 1 · pain hook · pinned · day 0
  • Post 2 · product mechanic · day 5-7

All four posts are 100% verbatim from the locked messaging system, with only "Summer 2026" added as launch context. If the messaging system updates, these posts update first.

Source-of-truth principle

Every line in every post traces to a locked anchor in docs/positioning/messaging-system.html. If you ever feel tempted to write something different on X, edit the messaging system first, then propagate. Never invent X copy in isolation.

This is the same discipline the homepage, /for-ai-work/, and /values/ follow. X is just another surface. Consistency across surfaces is what makes a brand feel like one product instead of many.

Posting schedule

When Post Action
Day 0 Post 1 · pain hook Post and pin to profile. Set bio to the locked X bio (see below).
Day 5-7 Post 2 · mechanic Post during 10am-2pm ET on Tue/Wed/Thu. Reply to any comments on Post 1 first.
Day 10-14 Post 3 · AI memory Post 3-5 days after Post 2. If Post 2 gained traction (5+ replies, 50+ impressions), wait the longer end of the window.
Day 15-21 Post 4 · depth Post when Post 3 has cooled. This one is the Obsidian/PKM hook; lands well after the AI-memory crowd has engaged.
Day 22+ Replies + engagement only Don't post a fifth positioning tweet. Engage with niche threads (Obsidian, MCP, voice tools) and post shipping screenshots if any. New positioning copy waits for the launch announcement.
Don't batch-schedule via Buffer / Hypefury. Founders posting from a scheduled queue read as inauthentic. Post each one manually. The 4-7 day gap matters — it gives each post its own attention window and lets you react to engagement on the previous one.

Time of day

  • Best window: 10am - 2pm ET (peaks at ~12pm). Catches both NA East and EU late-afternoon.
  • Best days: Tuesday, Wednesday, Thursday. Avoid weekends (low tech-audience engagement).
  • Worst: Friday after 3pm ET, Saturday morning, Sunday evening.

These windows matter less when the account has few followers because there's no algorithmic distribution. But once a tweet gains some early traction, the time of day affects how quickly X's algo decides whether to broadcast wider. Default to good timing even pre-launch.

Between-post engagement

The 4-7 day gap between scheduled posts is when most of the X work actually happens. Fill those days with:

  • Replies in the niche. Search X for terms like obsidian voice notes, MCP server, AI memory tools, local whisper, wispr flow. Reply to threads where ThoughtMic is genuinely useful context. No pitching. Just be a knowledgeable presence.
  • Shipping screenshots / progress. When you ship something visible (a new page, a feature, a comparison), post a screenshot with a one-line founder caption. These are not positioning copy and don't need to draw from the messaging system. Voice-y is fine here.
  • Quote-retweets of relevant news. If Obsidian, MCP spec, or local-AI ecosystems ship something newsworthy, QRT with a one-line take. Adds context, signals you're paying attention.
  • Reply to engagement on your own posts. If anyone replies, like, or QRTs the four scheduled posts, respond personally. Pre-launch engagement is rare and high-value.

Do NOT, between scheduled posts:

  • Post additional positioning copy (saves it for launch announcement)
  • Quote-tweet competitors with critique (the messaging system bans defensive naming)
  • Tweet about unrelated topics (politics, hot takes, random philosophy). The pre-launch profile should read as one focused thing.

Things we will NOT post

The same guardrails as messaging-system.html#dont-say apply here. Restating because X is a tempting place to slip:

  • No em-dashes (—) or en-dashes (–). Use periods, commas, line breaks, or short sentences. X auto-converts straight quotes but does NOT auto-convert dashes; what you type is what posts.
  • No "AI-powered" / "AI-driven".
  • No "revolutionary" / "game-changing" / "magical" / "delightful".
  • No naming competitors defensively. Wispr Flow, Superwhisper, Mem, Reflect. Comparison pages on the site name them once each; brand-level X posts do not.
  • No cloud provider names. Cerebras, Groq, Fireworks, OpenAI, Anthropic. The product backend uses these but the X account doesn't read like a provider press release.
  • No unvalidated habit claims. "Your daily routine", "every morning you...", "your weekly review habit" — all hypotheses until real users demonstrate them.
  • No "simply" or "just". Patronizing.
  • No "second brain" as a primary descriptor. Tiago Forte's framing; reference in passing is fine but don't lead with it.
  • No "transform your workflow" / "10x your productivity". Productivity-marketing slop.
  • No replying with the product pitch in unrelated threads. Be useful first, brand last.
