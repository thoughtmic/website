What this doc is. A layered messaging architecture for thoughtmic.com proposed during the 2026-05-19 positioning conversation. Read this end-to-end before approving any index.html changes. Once layers are locked, the verbatim implementation prompt lives in docs/positioning/IMPL-KICKOFF.md.

Summary

The current homepage is closer to right than it looks. The hero already does "voice-to-vault" without leading with AI. The compat strip already names Claude and MCP. The privacy section is strong. The three pillars are Obsidian-friendly.

The gaps are three:

  1. No dedicated section for the AI-augmented work persona (the founder's own use case, surfaced 2026-05-19, see project_ai_memory_wedge_2026_05_19.md).
  2. No /values page articulating the seven "we will" commitments that turn AI from a product anxiety into a bounded utility promise.
  3. The current copy makes some claims about downstream value (graph growth, MCP querying) that aren't yet validated by real users, see project_validation_gap_2026_05_19.md. A few sentences need tightening to land "this is what the product does" instead of "this is what users will eventually feel."

This plan does not propose a pivot. It proposes a layered addition: keep the Obsidian-native hero, add a self-selecting capability section for AI-augmented work, and ship a values page that names the AI-as-utility stance directly.

Layer 4: Values page New page at /values

Job to be done: Concrete commitments, not predictions. Says how we will behave. Doesn't require validation of user outcomes. Reflect's /our-values page is the closest reference. Trust foundation for both P1 and P2.

The seven commitments

1. Your spoken thoughts belong to you.

They live in your vault as Markdown files on your machine. Not in OpenAI's chat history, not in Anthropic's logs, not in our cloud. Yours.

2. Memory across every AI you use.

Whichever assistant you use today or next year, your captured thinking stays portable. MCP is the standard; we serve it locally so any compatible client can read your vault.

3. We will never lock your thoughts into a proprietary format.

You'd own these files even if ThoughtMic disappeared tomorrow. That's the test we hold ourselves to.

4. We use AI as a utility, never as the product.

Speech transcription runs locally on your Mac. Rephrase runs in the cloud, on the text you spoke, in milliseconds. AI does the narrow jobs you trigger. It doesn't read your vault. It doesn't chat with you. It doesn't own your data layer.

5. We will be searchable, by you and by any AI you authorize.

Through MCP, in your own time, on your own terms. Not as an upsell. Not behind a flag.

6. Your notes outlive your subscription.

Cancel ThoughtMic anytime. Your Markdown files stay where they are, fully readable and editable in any editor that opens text. The product can end; your notes don't.

7. We will not build lock-in integrations.

Granola, Otter, Zoom, Notion, Linear can have their own value. We connect through Markdown and MCP. If you leave us, you leave with everything.

Risks & cuts

  • Risk: The new /for-ai-work/ page reads as scope creep. Cut: drop the page; keep only the homepage callout that hints at the use case; let the values page do the trust work.
  • Risk: Sub-hero refinement loses SEO copy that's already ranking. Mitigation: P3 is the lowest-priority phase; defer until P1 / P2 land and SEO impact can be checked against Search Console data.
  • Risk: Values page reads preachy. Mitigation: review the seven commitments after a 24-hour cool-off; cut to five if any feel forced.
  • Risk: Validation gap still exists when launch ships 2026-06-16. Mitigation: recruit 3-5 real P2 users in next 4 weeks (separate workstream); use their language to revise /values in v1.1.
  • Risk: Google still shows "Did you mean: thoughtmix" at launch. Mitigation: SEO follow-up PR (Organization schema + Knowledge Graph submission); brand-name recognition is a months-long backlinks problem and won't fully resolve before launch.
  • Cut order if time slips: P4 (blog post) then P3 (sub-hero tighten) then P1 (AI work page + callout) then P2 (values page). Values page is the most load-bearing for trust; ship it even if everything else cuts.

SEO follow-up workstream Separate PR after this one

Search "thoughtmic" on Google as of 2026-05-20 returns only the homepage, no subpages, no favicon, with "Did you mean: thoughtmix" as a typo suggestion. Three distinct problems require separate fixes:

  1. Brand-name recognition. Google doesn't know "thoughtmic" is a brand yet. Fix is Knowledge Graph submission + consistent backlinks from credible sources. Months-long; backlinks accumulate over launch + post-launch outreach.
  2. Subpages not indexed / linked. The new pages from this PR (/values/, /for-ai-work/) add two more indexable surfaces. The SEO PR adds internal links from the homepage to /vs/*, /guides/*, /values/, /for-ai-work/, /privacy/; verifies sitemap.xml has them all; submits sitemap in Google Search Console.
  3. No favicon / logo in search. Add Organization + logo schema to index.html (separate from existing SoftwareApplication). Add BreadcrumbList on subpages.

This is a separate PR (proposed branch: feat/seo-organization-schema-internal-linking) so it doesn't gate on positioning copy approval. It can run in parallel once the positioning PR merges.
